# Code of Conduct

VoxelForge is committed to maintaining a respectful, constructive, and welcoming environment for everyone interacting with the project.

This Code of Conduct applies to all project spaces, including GitHub issues, discussions, community forums or chats, and any communication related to VoxelForge. By participating, you agree to follow these guidelines.

---

## Our Principles

| Principle | Description |
|-----------|-------------|
| **Respect** | Treat other users, contributors, and maintainers with courtesy and professionalism. |
| **Constructive Collaboration** | Feedback, feature suggestions, and issue reports should be helpful and constructive. |
| **Curiosity and Creativity** | VoxelForge exists to support creativity in Minecraft structure design and voxel-based modelling. |
| **Integrity** | Participants should communicate honestly and act responsibly. |

---

## Expected Behaviour

- Communicate respectfully
- Provide clear and constructive feedback
- Stay on-topic in discussions
- Respect the project's licensing and contribution policies
- Follow applicable platform rules (e.g. GitHub community guidelines)

---

## Unacceptable Behaviour

The following will not be tolerated:

- Harassment, threats, or abusive language
- Personal attacks or insults
- Hate speech or discrimination
- Spam or unrelated promotional content
- Sharing or redistributing proprietary code without permission
- Attempting to bypass licensing restrictions
- Disrupting community discussions or project spaces

---

## Responsible Use

VoxelForge is intended for creative and constructive use only.

### Prohibited Content

Users must not generate, distribute, or promote:

- Sexual content involving minors
- Exploitative or abusive material
- Content intended to harass, threaten, or harm individuals or groups
- Content that violates applicable laws or regulations

### Prohibited API / Service Abuse

- Overloading, disrupting, or performing denial-of-service attacks against external services
- Circumventing API rate limits or usage restrictions
- Attempting to reverse engineer or attack AI infrastructure
- Using automated tools to intentionally abuse external systems

### Prohibited Security Abuse

- Exploiting vulnerabilities
- Conducting denial-of-service attacks
- Distributing malware or malicious software

---

## AI Output Disclaimer

VoxelForge uses generative AI to create 3D structures from text prompts. Because of the nature of generative AI:

> ⚠️ Outputs may be unpredictable, inaccurate, or occasionally unintended. Users are responsible for reviewing generated outputs before using or sharing them. VoxelForge makes no guarantees regarding the accuracy, quality, or safety of AI-generated content.

---

## User Responsibility

Users are responsible for how they use and distribute structures generated by VoxelForge. All generated content must:

- Comply with applicable laws and regulations
- Not infringe on intellectual property rights
- Not violate platform or server rules where the structure is used

VoxelForge is not responsible for how users choose to use generated structures.

## Contributing

VoxelForge currently operates under a **restricted contribution model**. Pull requests and direct code contributions are not accepted without explicit permission from the project owner.

Community members are encouraged to contribute through:

- 🐛 [Issue reports](../../issues)
- 💡 Feature suggestions
- 💬 Feedback and discussions

Unauthorized code submissions may be declined or closed.

---

## Reporting Violations

If you encounter behaviour that violates this Code of Conduct:

- Open a repository issue (if appropriate)
- Contact the project maintainer directly

Reports will be reviewed and handled at the discretion of the project maintainer.

---

## Enforcement

The project maintainer reserves the right to:

- Remove comments or discussions
- Lock threads or issues
- Restrict or ban users from interacting with the project
- Revoke access to project resources

Enforcement decisions will be made to maintain a respectful and productive project environment. Participation is a privilege and may be revoked if these guidelines are not respected.
