# Security Policy

VoxelForge takes the security of its software, infrastructure, and users seriously. If you discover a security vulnerability, please report it responsibly so it can be addressed promptly.

---

## Supported Versions

VoxelForge is currently under active development. Security fixes are applied to the most recent version only. Older builds or experimental versions may not receive security updates.

| Version | Supported |
|---------|-----------|
| Latest  | ✅ Yes |
| Older builds | ❌ No |

---

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities privately by contacting the project maintainer directly. Include as much detail as possible:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The affected version of the software
- Proof-of-concept code or screenshots (if available)
- The potential impact of the vulnerability

> Please avoid publicly disclosing a vulnerability before it has been reviewed and addressed.

---

## Responsible Disclosure

VoxelForge follows a responsible disclosure approach. Security researchers and users are asked to:

- Report vulnerabilities privately
- Allow reasonable time for investigation and remediation
- Avoid exploiting the vulnerability beyond what is necessary to demonstrate the issue
- Avoid accessing or modifying data that does not belong to you

Public disclosure may be coordinated once the vulnerability has been fixed.

---

## Out-of-Scope Reports

The following are generally considered out of scope:

- Issues in third-party libraries or services not controlled by VoxelForge
- Social engineering attacks
- Denial-of-service attacks against project infrastructure
- Reports without sufficient detail to reproduce the issue

---

## Prohibited Security Testing

The following activities are strictly prohibited:

- Performing denial-of-service attacks against services used by VoxelForge
- Attempting to compromise infrastructure or external APIs
- Automated vulnerability scanning against hosted services without permission
- Accessing systems or data without authorisation

Security testing must not disrupt development infrastructure or external service providers.

---

## Security Updates

If a vulnerability is confirmed, the project maintainer may:

- Patch the vulnerability in a future release
- Issue a security advisory
- Update documentation or configuration guidance

Users are encouraged to keep their installation up to date to receive security fixes.

---

## No Bug Bounty

VoxelForge does not currently offer a bug bounty program or financial rewards for vulnerability reports. However, valid reports that help improve the security of the project are genuinely appreciated.

---

## Disclaimer

VoxelForge is provided **"as is"** without warranty of any kind. While reasonable efforts are made to maintain security, users are responsible for deploying and using the software in a secure environment.
